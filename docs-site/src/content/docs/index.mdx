`ado-aw` is a compiler for teams that want to define Azure DevOps agentic pipelines in a more human-friendly format. You write a markdown file with YAML front matter that describes the agent, tools, permissions, and schedule, and `ado-aw` compiles it into pipeline YAML that Azure DevOps can run.

## Built for safer agentic automation

Every compiled pipeline follows a three-stage model:

1. **Agent** — runs in a network-isolated sandbox with a read-only token and produces safe-output proposals.
2. **Detection** — inspects those proposals for prompt injection, secret leaks, and other threats.
3. **Execution** — applies approved outputs with a separate write-capable token that the agent never sees.

## Key features

- **Safe outputs** for controlled write actions such as pull requests, comments, and work items
- **Network isolation** for Stage 1 agents running inside the AWF sandbox model
- **Tool allow-listing** so agents only get the tools and hosts you explicitly permit
- **MCP support** for SafeOutputs, Azure DevOps integrations, and additional MCP-based extensions
- **Natural-language authoring** that keeps pipeline intent readable in markdown source files

Start with the quick start guide to create your first agent file, compile it, and add the generated pipeline to Azure DevOps.
